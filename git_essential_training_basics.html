  4. Git Concepts and Architecture
    • The Three Trees
      Repository, Staging Index, Working
      - checkout Repo -> Working
      - add Working -> Staging
      - commit Staging -> Repo
    • Git workflows
    • Hash values (SHA-1)
      git track changes with hash codes
      - is a checksum of the change set
      each change is a snapshot
      - can be a single change or group of changes
      checksum is unique to the changes in the check-in
      - any change would change the checksum
      checksum (SHA value) includes changes, parent SHA, author and message.
      - changing any of these changes the SHA value.
    • The HEAD pointer
      always points to the tip of current branch
      ls -la .git
      cat .git/HEAD
      cat .git/refs/head/master
      - contents of .git/refs/head/master will be the SHA of the check-in for HEAD check-in.
  5. Make Changes to Files
    • Add files
      git add filename {filename*}to add specific files
      git add .to add all the files in a folder
      git commit -m "commit message."to commit the changes
    • Edit files
      after modify/edit file(s), same add/commit commands as for adding files.
      - adding changes is called staging
    • View changes with diff
      git diffdiff changes between working and staging.
    • View only staged changes
      git diff --stageddiff changes staging and repo.
      - --cached is the same as --stage
    • Delete files
      git rm filenamethe delete is staged
      git commit -m "commit message."commit change to repo
    • Move and rename files
      rename a file
      mv filename1 filename2however you choose to rename a file
      git add filename2
      git rm filename1
      git commit -m "commit message."commit change to repo
      move a file
      git mv filename1 filename2file in same location
      git mv filename path/filenamefile to a different folder
      git commit -m "commit message."commit change to repo

    • In this section the gitbash cmd window in the tutorial showed a differnt command(s) to undo/revert changes in the varisous repos.
    • Undo working directory changes

    • Unstage files
      highlight
      note

      Can only amend to the last commit.
      git commit --amend -m message
      - same command to amend actual changes and to change just the commit message.
    • Retrieve old versions
      git checkout SHA -- filename
    • Revert a commit

  8. Ignore Files
    • Use .gitignore files
      project/.gitignore
      - list of rules to determine which files are ignored
      - changes made to files in .gitignore will be ignored
      contents of .gitignore
      - file names
      - *?[aeiou][0-9]
      *.txtall .txt files
      logs/*.txtall .txt files in logs dir
      *.php
      !index.php      		all .php files but not index.php
      build/logs/all files in build/logs dir
      logs/*.log.[0-9]*.log.[0-9] in logs dir
      - # for comments
    • Ideas on what to ignore
      (Template .gitignore files available at https://github.com/github/gitignore)
      Compiled source code
      - binaries
      Packages and compressed files
      Logs and databases
      OS generated files
      User-uploaded files
      - images, PDF's, videos, etc.
    • Globally ignore files
      git config --global core.excludesfile ~/.gitignore_globalset excludes for all repo's and projects
      global ignore is user specific.
      probably better to have it per project.
    • Ignore tracked files
      Add file(s) to .gitignore.
      git rm --cached filenamelocal copy exists but changes not tracked
      To track again just remove from the .gitignore.
    • Track empty directories
      create a .filename file in the empty directory.
      - suggested filename is .gitkeep
      add and commit the empty directory
      git ls-tree headlist files being tracked
