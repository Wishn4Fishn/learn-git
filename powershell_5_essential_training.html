Up and Running with PowerShell 5

  • Course Description
    • PowerShell is the must-have tool for Windows system administrators. Jump-start your career, and your IT environment, by getting up and running with the latest version, PowerShell 5. Jason Helmick shows you how to get a hand from the help system, extend PowerShell with modules and snap-ins, harness the power of the PowerShell pipeline, and script and automate common administrative tasks. Plus, learn how to take advantage of one of PowerShell's most exciting features, remote management, which can put thousands of machines at your beck and call.

  2. Discoery and Getting Help
    • Updating your most important resource: Help
      By default PowerShell does not ship with help. You need to update it.
      Update-Help -Force
    • Discovering commands
      Get-Help
      - search all the help systems not just the cmdlets.
      help or man
      - these will pause after each screen and not just scroll through everything.
      Get-Verb
    • Exploring help options
      Get-Help <cmdlet> -Detailed
      - Detailed help with examples.
      Get-Help <cmdlet> -Examples
      - Just the examples.
      Get-Help <cmdlet> -Full
      - Detailed help with additional properties.
      Get-Help <cmdlet> -ShowWindow
      - Full help in a pop-up dialog.
      Get-Help <cmdlet> -Online
      - Useful on older servers running PowerShell v2 or older.
    • Understanding cmdlet syntax
      Review SYNTAX section of the detailed help.
      Example:
      SYNTAX
         Get-Service [-ComputerName <System.String[]>] [-DependentServices]
         -DisplayName <System.String[]> [-Exclude System.String[]>] [-Include
         <System.String[]>] [-RequiredServices] [<CommonParameters>]

         Get-Service [-ComputerName <System.String[]>] [-DependentServices] [-
         Exclude <System.String[]>] [-Include <System.String[]>] [-InputObject
         <System.ServiceProcess.ServiceController[]>] [-RequiredServices]
         [<CommonParameters>]

         Get-Service [[-Name] <System.String[]>] [-ComputerName <System.String[]>]
         [-DependentServices] [-Exclude <System.String[]>] [-Include
         <System.String[]>] [-RequiredServices] [<CommonParameters>]

  4. Using the Power of the Pipeline
    • Understanding the purpose of the pipeline
      Use "|" to pass the output from one command to another command.
      Get-GetService -Name bits | Stop-Service
    • Working with files and printers
      Out-File to the output to a file.
      Get-Content = type = cat
    • Working with CSVs
      To see CSV related cmdlets
      Get-Help *csv*
      PowerShell by default Export-Csv puts in TYPE info as the first line of the .csv.
      - this may cause problems wioth other apps.
      - there is a parameter (-NoTypeInformation) that surpresses adding the TYPE info.
      - Output ends with Export-Csv.
      Convertto-Csv same as Export-Csv but it does not write it to a file so CSV output can be passed to an additional command.
    • Working with XML
      To see XML related cmdlets
      Get-Help *xml*
    • Storing data as a webpage
      To see HTML related cmdlets
      Get-Help *html*
      Example:
      Get-EventLog -LogName System -Newest 2 -EntryType Error | ConvertTo-Html -Title "Windows Errors" -Body (Get-Date) -PreContent "<p>Generated by IT</p>" -PostContent "For more info check the full log" | Out-File C:\error.html

  5. Getting More out of Objects
    • Understanding the importance of objects
      Use Get-Member to get the details (methods, properties, etc.) of an object.
      Example:
      - <CmdLet> | gm
      - gm is the alias for Get-Member
      MemberType:
      - *Property*: we can get information about the object from this
      - Method: something the object can do
    • Selecting the information you want
      To view just the properties that you want to see:
      - <CmdLet> | Select-Object -Property [String[]]
      - select is the alias for Select-Object
      Format-Table to change how output is displayed.
      - Get-Help *format* to see other formatting CmdLets
    • Sorting the information
      Sort-Object -Property String[] to sort output.
      - -Asending
      - -Desending
    • Customizing your own properties
      Select-Object adds the property quered to the object.
      Examples:
      Get-Service -Name bits | Select-Object -Property name, @{name="ServiceName";expression={"hello"}}

      Name          ServiceName
      ----          -----------
      bits          Hello

      Get-Service -Name bits | Select-Object -Property name, @{name="ServiceName";expression={"$_.name"}}

      Name          ServiceName
      ----          -----------
      bits          bits

      Get-Service -Name bits | Select-Object -Property @{name="ServiceName";expression={"$_.name"}}, status

      ServiceName          Status
      -----------          ------
      bits                 Stopped

      - name and expresstion can be abrviated as n and e.
      - in original PowerShell name was called label. can see that abrviated as l. don't confuse with a 1 (one).
      This will probably be handy one day. To get free disk space:
      Get-WmiObject -Class Win32_Logicaldisk -Filter "DeviceID='c:'" | Select-Object -Property DeviceID, @{name="FreeGB";expression={$_.FreeSpace / 1gb -as [int]}}, status

      DeviceID          FreeGB
      --------          ------
      C:                35

    • Filtering specific data
      Use Where-Object -FilterScript{} to filter data.
      - -FilterScript{} is positional so it is typically not shown in internet examples.
      Get-Help *operators* to see the help on available operators.
      -eq is equals. -ceq is case-sensitive equals.
      Write filters to trim as much as possible first.
    • Working with methods
      Methods are something the object can do.
      <CmdLet>.<Method>()

    • Using parameter attributes
      use param in trhe script to pass vairables to the script.
      - by using param and strongly type'ing the variables, this will allow Get-Help to show what is expected when Get-Help is done on the script.
    • Using command-based help
      Add something like this at the begining of the .ps1 file for default PS help on your script.
      <#
      .Synopsis
      This is a brief comment
      .Description
      This is a longer comment
      .Parameter CompouterName
      This is the name of a remote computer
      .Example
      Connecting to local computer
      DiskInfo -ComputerName localhost
      .Example
      Connecting to remote computer
      DiskInfo -ComputerName rqa3-sql02
      #>
      - after the #> to end the block comment. there can be no more than one blank line before the firsdt line of code.
      Get-Help about_Comment_Based_Help for more on what can be done with the help section.

  7. Scalable Management with PowerShell Remoting
    • Introducing PowerShell remoting
    • Enabling PowerShell remoting
      Enabled on Server 2012 and above
      Enable-PSRemoting
      - if you are sitting at the server.
      Firewall requirement: TCP port 5985
    • Managing one-to-one operations
      when working on a single machine.
      Enter-PSSession -ComputerName #&l;computername#>
      - kind of like an ssh session.
    • Scaling out with one-to-many operations
      Use Invoke-Command -ComputerName #&l;computername(s)#> to send commands to remote PS session(s) on computername.
      - session dies when command is completed.
      New-PSSession -ComputerName #&l;computername(s)#> to start a persistant PS session on the remote computer.
      - Invoke-Command -Session #&l;sessionname(s)#> to send commands to the sessions.

      - recommended to install, do what the GUI is needed for and then un-install.

      - recommends putting these in $profile if going to use them regualarly.
